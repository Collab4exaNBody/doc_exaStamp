# **Domain & Regions**

This section covers how the simulation domain itself is defined, and how to carve out named spatial regions within it — used to populate the system with particles, restrict analysis to a subvolume, or assign values to a subset of the grid.

- [**Simulation Domain**](domain/index.md) — the `domain` operator: cell size, grid dimensions, bounds, periodic/mirror boundary conditions and the physical/grid space transform
- [**Spatial Regions**](regions/index.md) — the `particle_regions` operator and its geometric shapes (boxes, planes, cylinders, spheres, cones), boolean region expressions, and the operators that consume them
